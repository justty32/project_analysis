# mylang 語言規格說明書 (Language Specification)

本文檔定義了 mylang 作為 **圖靈完備的 C++ 前置處理器** 的完整規格。其核心設計目標是透過 Lisp 風格的語法與強大的元編程能力，生成高效的 C++ 原始碼。

## 1. 核心結構與理念
- **二維結構**：原始碼被解析為「行」與「單詞」的二維 List。每一行預設是一個 List，並依序執行。
- **縮排規則 (Indentation)**：
  - 若一行的開頭是 Tab (無論幾個都視為一個)，則該行層級深一個 List。
  - 被小括號包住的單詞（不論跨行與否）皆屬於同一行。
- **C++ 代碼生成**：若執行的 List 返回字串，則該字串會被附加到生成的 C++ 原始碼中。
- **元編程環境**：整個環境是一個全域的 Dict（類似 Lua 的 `_env` 或 Python），用於儲存元求值階段的符號與函數。
- **AST 規格**：語法解析（Parser）階段後的輸出為純粹的 `list<symbol>`。此階段不進行型別辨識（如數字或字串內容均視為符號 Atom），所有型別轉換與求值均推遲至元求值（Evaluator）階段。

## 2. 詞法規則 (Lexical Rules)

### 2.1 優先順序 (Priority)
`\` (跳脫) > 註解 > 縮排 > `"` (字串) > `'` (引號) > `()` (List) > `{}` (Dict/Set) > `[]` (Array) > 單詞 (Atom/Symbol)。

### 2.2 轉義與特殊符號
- **跳脫字元 (`\`)**：
  - 用於在 Symbol 中包含空格或引號，例如 `\'atom` -> `'atom`, `atom\ haha` -> `atom haha`。
  - **續行**：若 `\` 位於行末，則下一行會被串接到當前行，並在接合處加上一個空格。
- **括號**：除非被跳脫，否則 `()`, `[]`, `{}` 會被視為單獨單詞切分。例如 `(word)` 切分為 `(`, `word`, `)`。
- **字串 (`"`)**：`"XXX"` 轉為 `(str XXX)`。優先級僅次於註解，結尾引號可省略。

## 3. 語法糖與擴展 (Syntax Sugar)

### 3.1 引號擴展規則 (`'`)
依據字串首尾有無 `'` 決定嵌套結構：
- **前向嵌套** (首有尾無)：`'A` -> `(quote A)`, `'A'B` -> `(A B)`, `'A'B'C` -> `(A (B C))`。
- **後向嵌套** (首無尾有)：`A'` -> `(A)`, `A'B'` -> `(A B)`, `A'B'C'` -> `((A B) C)`。
- **扁平 List** (首尾皆無)：`A'B` -> `(A B)`, `A'B'C` -> `(A B C)`。
- **對稱規則** (首尾皆有)：視為扁平 List。`'A'B'C'` -> `(A B C)`。
- **型別前綴**：`'int'123` -> `(int 123)`。
- **特殊情況**：`''` 視為 `()`；`''A` 視為 `'A`。

### 3.2 容器語法糖
- **Array (`[]`)**：`[a, b, c]` -> `(arr a b c)`。空格無效，以 `,` 分隔。
- **Dict/Set (`{}`)**：
  - **Dict**：`{a: 1, b: 2}` -> `(dict (quote (a 1)) (quote (b 2)))`。以 `:` 分隔鍵值。
  - **Set**：`{a, b, c}` -> `(set a b c)`。
  - 類型判定：取決於是否有給予 value (即有無 `:`)。

## 4. 核心系統函數

- **`get`**：檢索但不對結果求值。
  - `(get 'sym)`：從環境中取值。
  - `(get list index)`：取 List/Array 元素。
  - `(get dict key)`：取 Dict 鍵值。
- **`exec`**：顯式對 Value 求值（Symbol, List, Dict）。
- **`make`**：統一入口。如 `(make "int" 123)`, `(make "std::list<int>" 1 2 3)`。
- **`def`, `def_func`, `def_macro`**：遵循 Lisp/Lua 慣例。

## 5. 運行時與 C++ 映射
- 使用 `std::any` 或 `std::variant` 實現 List 的動態型別。
- Array/Set/Dict 鼓勵使用同型別以提升效率。
- 提供 `to_symbol` 等內建工具。
