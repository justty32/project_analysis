# mylang 語言規格說明書 (Language Specification)

本文檔定義了 mylang 作為 **圖靈完備的 Lisp 風格腳本語言** 的完整規格。其核心設計目標是透過 Lisp 風格的語法與強大的元編程能力，提供一個輕量、靈活且易於擴展的執行環境。

## 1. 核心結構與理念
- **二維結構**：原始碼被解析為「行」與「單詞」的二維 List。每一行預設是一個 List，並依序執行。
- **縮排規則 (Indentation)**：
  - 若一行的開頭是 Tab (無論幾個都視為一個)，則該行層級深一個 List。
  - 被小括號包住的單詞（不論跨行與否）皆屬於同一行。
- **求值模型**：腳本由評估器（Evaluator）逐行執行，每一行執行後會返回一個 `Value`。
- **執行環境**：整個環境是一個全域的 Dict（類似 Lua 的 `_env` 或 Python），用於儲存符號、變數與函數。
- **AST 規格**：語法解析（Parser）階段後的輸出為純粹的 `list<symbol>`。此階段不進行型別辨識（如數字或字串內容均視為符號 Atom），所有型別轉換與求值均推遲至執行（Evaluator）階段。

## 2. 詞法規則 (Lexical Rules)

### 2.1 優先順序 (Priority)
`\` (跳脫) > 註解 > 縮排 > `"` (字串) > `'` (引號) > `()` (List) > `{}` (Dict/Set) > `[]` (Array) > 單詞 (Atom/Symbol)。

### 2.2 轉義與特殊符號
- **跳脫字元 (`\`)**：
  - 用於在 Symbol 中包含空格或引號，例如 `\'atom` -> `'atom`, `atom\ haha` -> `atom haha`。
  - **續行**：若 `\` 位於行末，則下一行會被串接到當前行，並在接合處加上一個空格。
- **括號**：除非被跳脫，否則 `()`, `[]`, `{}` 會被視為單獨單詞切分。
  - **字串 (`"`)**：`"XXX"` 轉為 `(str "XXX")`。優先級僅次於註解，結尾引號可省略。
    - **多行支援**：字串可以跨越多行，直到遇到未被轉義的結束引號。
    - **轉義字元**：支援 `\n` (換行), `\t` (製表), `\r` (回車), `\"` (雙引號), `\\` (反斜線)。
    - **字串內續行**：在字串內部使用 `\` 緊跟換行符，會將該換行符移除並合併前後內容。
    - **求值行為**：轉譯後的 `(str "XXX")` 將由內建 `str` Macro 處理，這允許在字串中嵌入其他表達式或進行拼接。


## 3. 語法糖與擴展 (Syntax Sugar)

### 3.1 引號擴展規則 (`'`)
依據字串首尾有無 `'` 決定嵌套結構：
- **前向嵌套** (首有尾無)：`'A` -> `(quote A)`, `'A'B` -> `(A B)`, `'A'B'C` -> `(A (B C))`。
- **後向嵌套** (首無尾有)：`A'` -> `(A)`, `A'B'` -> `(A B)`, `A'B'C'` -> `((A B) C)`。
- **扁平 List** (首尾皆無)：`A'B` -> `(A B)`, `A'B'C` -> `(A B C)`。
- **對稱規則** (首尾皆有)：視為扁平 List。`'A'B'C'` -> `(A B C)`。
- **型別前綴**：`'int'123` -> `(int 123)`。
- **特殊情況**：`''` 視為 `()`；`''A` 視為 `'A`。

### 3.2 容器語法糖
- **Array (`[]`)**：`[a b c]` -> `(arr a b c)`。元素間以空格分隔（不使用逗號）。
- **Dict/Set (`{}`)**：
  - **Dict**：`{a: 1 b: 2}` -> `(dict (quote (a 1)) (quote (b 2)))`。鍵值對間以空格分隔。
  - **Set**：`{a b c}` -> `(set a b c)`。

### 3.3 元編程語法 (Meta-programming)
- **Backquote (`` ` ``)**：`(backquote expr)`。進入模板模式，允許內部求值。
- **Unquote (`,`)**：`(unquote expr)`。在 Backquote 內部對表達式求值並嵌入。
- **Unquote-Splicing (`,@`)**：`(unquote_splicing expr)`。對表達式求值，並將其列表/陣列內容展開嵌入。

## 4. 核心系統函數

- **`get`**：檢索但不對結果求值。
- **`exec`**：顯式對 Value 求值（Symbol, List, Dict）。
- **`make`**：資料結構構造函數。
- **`def`, `def_func`, `def_macro`**：定義變數、函數與 Macro。
- **`if`, `lambda`, `let`**：流程控制與作用域管理。

## 5. 運行時特性
- 支援閉包（Lexical Scoping）。
- 動態型別系統，延遲型別轉換。
- 易於嵌入 C++ 應用程序。
