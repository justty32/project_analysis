# mylang 語言規格說明書 (Language Specification)

本文檔定義了 mylang 作為 **圖靈完備的 C++ 前置處理器** 的完整規格。其核心設計目標是透過 Lisp 風格的語法與強大的元編程能力，生成高效的 C++ 原始碼。

## 1. 核心結構與理念
- **二維結構**：原始碼被解析為「行」與「單詞」的二維 List。每一行預設是一個 List，並依序執行。
- **C++ 代碼生成**：若執行的 List 返回字串，則該字串會被附加到生成的 C++ 原始碼中。
- **元編程環境**：整個環境是一個全域的 Dict（類似 Lua 的 `_env` 或 Python），用於儲存元求值階段的符號與函數。

## 2. 詞法規則 (Lexical Rules)

### 2.1 優先順序 (Priority)
`\` (跳脫) > `//`, `/**/` (註解) > `"` (字串) > `'` (引號擴展) > `()` (List) > `{}` (Dict/Set) > `[]` (Array) > 單詞 (Atom/Symbol)。

### 2.2 轉義與特殊符號
- **跳脫字元 (`\`)**：
  - 用於在 Symbol 中包含空格或引號，例如 `\'atom` -> `'atom`, `atom\ haha` -> `atom haha`。
  - **續行**：若 `\` 位於行末，則下一行會被串接到當前行，並在接合處加上一個空格。
- **括號**：除非被跳脫，否則 `()`, `[]`, `{}` 會被視為單獨單詞切分。例如 `(word)` 切分為 `(`, `word`, `)`。
- **字串 (`"`)**：`"XXX"` 轉為 `(str XXX)`。結尾引號若省略則自動補齊。

## 3. 語法糖與擴展 (Syntax Sugar)

### 3.1 引號擴展規則 (`'`)
- **前向巢狀**：`'A` -> `(quote A)`, `'A'B` -> `(A B)`, `'A'B'C` -> `(A (B C))`。
- **後向巢狀**：`A'` -> `(A)`, `A'B'` -> `(A B)`, `A'B'C'` -> `((A B) C)`。
- **直接連接**：若頭尾皆無 `'`，則為扁平 List。`A'B` -> `(A B)`, `A'B'C` -> `(A B C)`。
- **對稱規則**：若前後皆有 `'`，則視為前後皆無（扁平 List）。`'A'B'C'` -> `(A B C)`。
- **連續引號**：`''` 為空 List `()`；`''A` 視為 `'A`。
- **型別前綴**：`'int32'123` -> `(int32 123)`。

### 3.2 容器語法糖
- **Array (`[]`)**：`[a, b, c]` -> `(arr a b c)`。空格不具分隔功能，僅以 `,` 分隔。
- **Dict/Set (`{}`)**：
  - **Dict**：`{a: 1, b: 2}` -> `(dict (pair a 1) (pair b 2))`。以 `:` 分隔鍵值，`,` 分隔對項。
  - **Set**：`{a, b, c}` -> `(set a b c)`。以 `,` 分隔元素。

## 4. 核心系統函數

- **`get`**：檢索函數，不對結果再求值。支援 Symbol（自動轉字串）、List/Array（索引取值）、Dict（鍵值取值）。
- **`exec`**：顯式對一個 Value 進行求值。
- **`make`**：統一的資料結構構造入口。
- **`def`, `def_func`, `def_macro`**：定義環境變數、函數與巨集。

## 5. 運行時機制
- **Macro**：參數不求值，回傳結果會被再次求值。
- **可執行 Dict**：若 Dict 的 `_metatable._exec` 存在，則該 Dict 可作為函數執行。
- **C++ 映射**：使用 `std` 容器實作。Array/Set/Dict 的 Key 傾向同型別優化；List 使用 `std::any`。
