# mylang 架構設計文件 (Architecture)

## 1. 系統流程 (System Flow)

1. **Lexer (詞法分析)**：
   - 處理最高優先級的 `\` (轉義與續行)。續行處會補上一個空格。
   - 過濾註解 `//`, `/**/`。
   - 解析字串 `"` (支持省略結尾)、引號 `'`。
   - 根據括號深度判定「邏輯行」。
2. **Parser (語法解析)**：
   - 將 Token 流轉換為 `Value` (AST)。
   - **語法糖轉譯**：
     - **引號擴展**：根據引號位置建立遞迴巢狀或扁平 List。
     - **容器初始化**：`[]` -> `(arr ...)`, `{}` -> `(dict ...)` 或 `(set ...)`。
3. **Meta-Evaluator (元求值)**：
   - 執行 Lisp 邏輯。若結果為字串，則生成對應的 C++ 原始碼。
4. **Code Generator (代碼生成)**：
   - 收集元求值的輸出，彙整為最終的 C++ 檔案。

## 2. 資料結構 (Data Model)
- **LIST**: `std::list<Value>` (核心求值單位)。
- **ARRAY**: `std::vector<Value>` (高效連續記憶體)。
- **DICT**: `std::map<Value, Value>` (關聯陣列)。
- **SET**: `std::set<Value>` (唯一集合)。
- **SYMBOL**: 原始單詞，映射至元環境的鍵。

## 3. 核心設計原則
- **前處理器導向**：mylang 的主要職責是生成 C++ 代碼，而非作為獨立的運行時環境。
- **Lisp 求值哲學**：除了語法糖外，一切皆為 List 求值。
- **優先級嚴格執行**：詞法分析階段必須嚴格遵守定義的優先級。
