# mylang 架構設計文件 (Architecture)

## 1. 系統流程 (System Flow)

1. **Lexer (詞法分析)**：
   - 處理最高優先級的 `\` (轉義與續行)。續行處會補上一個空格。
   - 過濾註解 `//`, `/**/`。
   - 解析字串 `"` (支持省略結尾)、引號 `'`。
   - **層級判定**：
     - 根據括號深度與行首 **Tab** 數量判定「邏輯行」。
     - 若行首有 Tab（不論數量），則該行視為深一層的 List。
2. **Parser (語法解析)**：
   - 將 Token 流轉換為純粹的 `list<symbol>` (AST)。
   - **AST 規格**：此階段不進行型別辨識（如數字或字串內容均視為符號 ATOM），所有型別轉換與求值均推遲至元求值（Evaluator）階段。
   - **語法糖轉譯**：
     - **引號擴展**：根據首尾有無 `'` 建立前向/後向巢狀或扁平 List。
     - **容器初始化**：
       - `[]` -> `(arr ...)`：僅以 `,` 分隔。
       - `{}` -> `(dict ...)` 或 `(set ...)`：僅以 `,` 或 `:` 分隔。
3. **Meta-Evaluator (元求值)**：
   - 執行 Lisp 邏輯。核心函數包括 `get` (檢索不求值)、`exec` (顯式求值)、`make` (統一構造)。
   - **求值前轉換**：在對 ATOM 求值時，若其內容符合數字型態，則動態轉換為 NUMBER 進行運算。
   - 整個環境為一個 Dict (類似 Lua 的 `_env`)。
   - 若求值結果為字串，則生成對應的 C++ 原始碼。
4. **Code Generator (代碼生成)**：
   - 收集元求值的輸出，彙整為最終的 C++ 檔案 (`output.cpp`)。

## 2. 資料結構 (Data Model)
- **LIST**: `std::list<Value>` (核心求值單位)。
- **ATOM**: `std::string` (AST 的基本組成元素，代表符號或字串內容)。
- **NUMBER**: `double` (僅在求值階段產生，用於數學運算)。
- **ARRAY**: `std::vector<Value>` (高效連續記憶體，由 `arr` 語法糖生成)。
- **DICT**: `std::map<Value, Value>` (由 `{k:v}` 語法糖生成)。
- **SET**: `std::set<Value>` (由 `{v1, v2}` 語法糖生成)。

## 3. 核心設計原則
- **前處理器導向**：mylang 的主要職責是生成 C++ 代碼。
- **Lisp 求值哲學**：除了語法糖外，一切皆為 List 求值。
- **Tab 縮排語法**：簡化巢狀 List 的書寫，根據行首 Tab 數量判定深度。
- **優先級嚴格執行**：詞法分析階段必須嚴格遵守定義的優先級：`\` > 註解 > 縮排 > `"` > `'`。
- **延遲型別化**：Parser 保持 AST 的純粹性，將型別決策權留給 Evaluator。
