# 數據管理與資源載入 - Advanced Strategic Command (ASC)

ASC 使用自定義數據管理系統來載入遊戲資源、單位定義及地圖數據。

## 資源載入 (`source/loaders.cpp`)

遊戲引擎通過多個專門的載入器載入資源：

- **影像載入器 (`source/loadimage.h`, `source/loadpcxc.cpp`)**: 支援 PCX 等常見格式及單位、地形與 UI 元素的自定義圖形。
- **音效載入器**: 處理音效與音樂的載入。
- **單位與地形類型載入器**: 從設定檔中讀取單位類型、建築與地形類型的定義。

## 文本文件解析 (`source/textfileparser.cpp`)

ASC 的許多數據存儲在具有自定義格式的人類可讀文本文件中。`TextFileParser` 及相關類別提供：

- **鍵值對解析**: 從文本文件中提取遊戲參數。
- **標記化數據 (Tagged Data)**: 支援使用標記（如 [UNIT] ... [/UNIT]）的結構化數據。
- **屬性求值**: 計算數據文件中的複雜表達式與變數。

## 數據存檔 (Data Archives)

ASC 可以將其數據封裝到存檔中以優化載入與組織。這些存檔通常位於 `data/` 目錄中。

## 地圖文件 (Map Files)

地圖文件 (`.map`) 包含遊戲地圖的狀態：

- **網格數據**: 每個六角格的地形類型與初始物件。
- **實體數據**: 地圖上的初始單位與建築。
- **玩家數據**: 勢力、同盟關係與初始條件。
- **腳本化事件**: 為該地圖定義的基於觸發器的邏輯。
