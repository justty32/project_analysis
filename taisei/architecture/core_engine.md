# 核心引擎架構

Taisei Project 的核心引擎使用 C 語言編寫，管理應用程序的生命週期、事件和狀態。

## 初始化和生命週期 (`src/main.c`)

應用程序啟動後會初始化核心子系統：
-   **日誌和錯誤處理**：標準控制台和文件日誌。
-   **VFS**：在訪問其他資源之前設置虛擬文件系統。
-   **SDL**：初始化 SDL (Simple DirectMedia Layer) 以進行平台抽象。
-   **Config**：加載用戶首選項。
-   **子系統**：初始化視頻、渲染器、音頻、遊戲手柄和資源管理器。

引擎使用「調用鏈（Call Chain）」機制 (`CallChain` 和 `CallChainResult`) 來管理主要遊戲狀態之間的轉換（例如：啟動畫屏 -> 主菜單 -> 關卡 -> 遊戲結束）。

## 事件循環 (`src/eventloop/eventloop.c`)

主循環處理 SDL 事件、更新遊戲邏輯並處理渲染。
-   **邏輯更新**：通常固定在 60 FPS（在 `global.h` 中配置為 `FPS`）。
-   **渲染**：每一幀都會發生，獨立於邏輯更新（儘管通常是同步的）。
-   **無頭模式 (Headless Mode)**：支持在沒有窗口或音頻的情況下運行，用於重新播放錄製的遊戲以驗證結果。

## 全局狀態 (`src/global.h`)

一個單一的 `Global` 結構體 (`extern Global global;`) 持有整個瞬時遊戲狀態，包括：
-   活動實體列表：彈幕 (`projs`)、粒子 (`particles`)、敵人 (`enemies`)、物品 (`items`)、激光 (`lasers`)。
-   玩家對象 (`plr`)。
-   計時信息 (`frames`)。
-   隨機數狀態 (`rand_game` 和 `rand_visual`)。
-   回放輸入/輸出狀態。

## 線程和任務管理 (`src/taskmanager.h`)

Taisei 實現了自己的多線程任務管理器來處理後台處理：
-   **工作線程**：根據系統的 CPU 核心數生成多個線程。
-   **基於隊列的執行**：任務提交到優先級隊列，並在線程可用時執行。
-   **確定性**：遊戲關鍵邏輯通常與多線程任務分開，以確保回放的確定性。

## 確定性和回放

確定性是一個核心設計目標。引擎通過以下方式實現：
-   **固定邏輯速率**：無論實際幀率如何，遊戲邏輯都以恆定速率更新。
-   **分離的隨機生成器**：`rand_game` 用於影響遊戲的決策（生成位置、彈幕軌跡）並記錄在回放中。`rand_visual` 用於不影響遊戲的效果（如粒子）。
-   **回放錄製**：錄製所有用戶輸入和初始種子，使引擎能夠通過「重新運行」邏輯來重現整個遊戲會話。
