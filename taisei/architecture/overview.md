# Taisei Project 架構總覽

Taisei Project 是一個使用 C 語言編寫的跨平台、開源東方風格彈幕遊戲引擎。它擁有專為處理彈幕遊戲典型的高密度彈幕和複雜移動模式而設計的自定義引擎。

## 核心架構支柱

1.  **基於協程的遊戲邏輯**：引擎使用先進的協程系統（基於 `koishi`）來處理遊戲實體（玩家、敵人、彈幕）。通過 C 宏實現的自定義 DSL（領域特定語言）允許開發者以命令式、線性風格編寫複雜的、感知幀數的邏輯。
2.  **不可知渲染層**：自定義渲染器 API 抽象了底層圖形後端。它支持 OpenGL 3.3、OpenGL ES 3.0 和 SDL_gpu，並包含一個用於無窗口執行（例如回放驗證）的空渲染器（null renderer）。
3.  **虛擬文件系統 (VFS)**：強大的 VFS 層抽象了文件系統訪問，支持從目錄、zip 文件和內存中透明加載，以及處理特定平台的路徑。
4.  **多線程任務管理**：雖然主要的遊戲邏輯通常是單線程（或基於協程）的，但後台任務管理器處理繁重的工作，如資源加載和解壓縮。
5.  **確定性回放系統**：引擎設計追求確定性，將影響遊戲的隨機數生成與純視覺隨機性分開，從而實現可靠的回放系統。

## 項目結構

- `src/`：核心引擎和遊戲源代碼。
    - `audio/`：音頻和音樂子系統。
    - `coroutine/`：協程和任務 DSL 實現。
    - `renderer/`：圖形抽象層和後端。
    - `resource/`：資源管理（紋理、模型、著色器等）。
    - `vfs/`：虛擬文件系統。
    - `stages/`：特定關卡的腳本和數據。
    - `plrmodes/`：玩家角色定義和射擊類型。
- `resources/`：遊戲資源（通過 VFS 打包）。
- `external/`：捆綁的第三方依賴。
- `scripts/`：構建和開發工具腳本。
