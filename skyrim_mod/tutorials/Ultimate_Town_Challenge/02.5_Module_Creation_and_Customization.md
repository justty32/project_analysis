# 02.5. 終極挑戰：模塊的製作與動態定制

在開始編寫代碼之前，你必須先擁有「合規」的模組（Modules/Prefabs）。本篇教學將分為兩部分：第一部分教你如何設計一個適合 C++ 拼接的模組，第二部分教你如何用插件動態修改生成的模組細節（如更換壁紙、添加裝飾）。

---

## 1. 模組設計指南 (Creation Kit 篇)

要讓 C++ 能夠完美拼裝，你的模組必須遵循以下規則：

### A. 坐標原點 (Pivot Point)
-   **規則**: 模塊的旋轉中心（原點 0,0,0）必須位於模塊的「入口邊緣」或正中心。
-   **原因**: C++ 透過坐標計算放置物體，如果模型原點不在邊緣，你計算拼接點時會面臨極其複雜的偏移補償。

### B. 預製導航網格 (Navmesh)
-   **步驟**: 在 CK 中為這個模塊單獨劃分 Navmesh。
-   **連通性**: 確保 Navmesh 的頂點剛好落在模型邊界的邊緣。這樣當兩個模組靠近時，引擎會自動將兩塊 Navmesh 縫合。

## 2. 透過插件修改模塊細節

一旦 `PlaceAtMe` 生成了模組，它就是一個獨立的 `RE::TESObjectREFR`。你可以對其進行深度定制。

### A. 動態更換貼圖 (Texture Swapping)
你想讓同一個房屋模組，有時候是木頭牆，有時候是石牆？

```cpp
void SwapTexture(RE::TESObjectREFR* a_ref, RE::FormID a_textureSetID) {
    // 獲取紋理集 (include/RE/B/BGSTextureSet.h)
    auto textureSet = RE::TESForm::LookupByID<RE::BGSTextureSet>(a_textureSetID);
    
    if (a_ref && textureSet) {
        // 使用 ExtraDataList 來更換模型材質
        auto extraTexture = new RE::ExtraTextureSet(textureSet);
        a_ref->extraList.Add(extraTexture);
        
        // 強制刷新 3D
        a_ref->Update3DModel();
    }
}
```

### B. 動態裝飾 (Procedural Detailing)

### C. 修改縮放比例 (Scaling)
讓同一個模組看起來像不同的建築。
```cpp
// 讓房子變大 20%

```

## 3. 關鍵 API 標註
-   **`RE::BGSTextureSet`**: 定義一組貼圖路徑。`include/RE/B/BGSTextureSet.h`
-   **`RE::ExtraTextureSet`**: 用於實體引用的材質覆蓋。`include/RE/E/ExtraTextureSet.h`
-   **`Update3DModel()`**: 當你修改了材質或模型後，必須調用此函數才能在遊戲中看到變化。

## 4. 總結
1.  **在 CK 中**: 確保模塊原點精確，Navmesh 邊緣對齊。
2.  **在 C++ 中**: 透過 `ExtraTextureSet` 實現一物多用，透過局部座標計算實現動態裝飾。

## 5. 原始碼參考與核心函數

### 核心頭文件
- `include/RE/B/BGSTextureSet.h` - 貼圖集合定義。
- `include/RE/E/ExtraTextureSet.h` - 實體材質覆蓋。
- `include/RE/T/TESObjectREFR.h` - 縮放與模型更新。

### 推薦使用的函數
- `RE::TESObjectREFR::SetScale(float a_scale)`：動態調整模塊大小。
- `RE::TESObjectREFR::Update3DModel()`：核心！應用材質或模型變更。
- `RE::ExtraDataList::Add(new RE::ExtraTextureSet(textureSet))`：動態更換外牆貼圖。
- `RE::TESForm::LookupByID<RE::BGSTextureSet>(FormID)`：獲取預定義的材質包。
