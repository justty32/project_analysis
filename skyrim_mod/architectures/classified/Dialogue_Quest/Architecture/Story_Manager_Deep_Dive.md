# 深度解析：故事管理器 (BGSStoryManager)

故事管理器（Story Manager）是 Skyrim 的「事件中心」，負責將遊戲中發生的零散行為轉化為有條理的任務與劇情。

## 1. 核心組件：Event Nodes (事件節點)
- **原始碼**: `include/RE/B/BGSStoryManager.h`
- **機制**: 引擎預設了幾十種「節點類型」，例如：
    - **`Kill Actor`**: 玩家殺死了一個 NPC。
    - **`Level Up`**: 玩家升級。
    - **`Pickpocket`**: 玩家扒竊被抓。
    - **`Quest Start`**: 另一個任務開始。

## 2. 決策樹 (The Tree Logic)
當一個事件發生（例如：玩家殺死了一個帶名字的 NPC），管理員會執行以下操作：
1.  **查找分支**: 從根節點向下掃描與 `Kill Actor` 相關的分支。
2.  **條件評估 (Conditions)**: 每個分支都有嚴格的條件。
    - “NPC 是否屬於某個家族？”
    - “玩家是否處於某個城鎮？”
3.  **衝突解決**: 如果有多個任務符合條件，管理員會根據任務的 `Priority` 選擇一個觸發，或並行觸發（取決於節點設置）。

## 3. 性能與緩存
故事管理器的掃描是極其高效的。它不會遍歷所有任務，而是預先為每個節點建立了一張「潛在任務表」。

## 4. C++ 介入：動態劇情注入
透過 SKSE，你可以繞過 ESP 的靜態限制：
- **自定義節點**: Hook 故事管理器的分發函數，當特定 C++ 事件發生時（如：玩家連續 5 分鐘沒動），模擬發送一個 `Idle` 節點來觸發特殊任務。
- **強行截斷**: 在故事管理器觸發任務前攔截它，防止某些原本會發生的動態任務（如：僱傭兵復仇）。

## 5. 核心類別原始碼標註
- **`RE::BGSStoryManager`**: `include/RE/B/BGSStoryManager.h` - 全局管理單例。
- **`RE::Condition`**: `include/RE/C/Condition.h` - 決定節點是否通過的核心判定類。
- **`RE::TESQuest`**: `include/RE/T/TESQuest.h` - 任務的載體。
