# VCMI_server 模組分析 (`server/`)

`VCMI_server` 是 VCMI 的權威來源。它負責維護遊戲狀態、驗證操作並將結果廣播到各個客戶端。

## 核心組件

### 1. 遊戲處理器 (`CGameHandler`)
- **上帝類 (God Class)**: 伺服器的核心，包含處理所有冒險地圖與戰鬥邏輯的方法。
- 負責：
    - 驗證玩家操作（英雄移動、攻擊、建設等）。
    - 執行隨機事件（隨機戰鬥結果、資源產出）。
    - 更新 `CGameState` 物件。
    - 透過 `IGameServer` 介面將變更發送給客戶端。

### 2. 伺服器主體 (`CVCMIServer`)
- 管理網路連接：監聽端口、接受新玩家。
- 控制遊戲迴圈：處理封包接收與分發。
- 管理遊戲設置與載入流程。

### 3. 專屬處理器 (`processors/`)
- 為了避免 `CGameHandler` 過於龐大，部分邏輯被抽離到專屬處理器：
    - **`NewTurnProcessor`**: 處理回合開始時的邏輯（資源產出、建築收益、生物產量）。
    - **`HeroPoolProcessor`**: 管理酒館中的英雄池。
    - **`PlayerMessageProcessor`**: 處理玩家間的聊天與消息。

### 4. 戰鬥引擎 (`battles/`)
- 伺服器端的戰鬥模擬，決定單位行動順序、傷害計算及 AI 行為。

### 5. 查詢系統 (`queries/`)
- 處理客戶端對伺服器數據的特定查詢，例如查詢一個生物的詳細屬性。

## 同步機制
VCMI 採用 **狀態同步 (State Synchronization)**：
1. 客戶端發送請求封包 (`NetPack`) 到伺服器。
2. 伺服器在 `CGameHandler` 中驗證請求是否合法。
3. 伺服器修改權威的 `CGameState`。
4. 伺服器將狀態變更封包廣播給受影響的客戶端。
5. 客戶端根據接收到的變更更新本地的 `CGameState`。
