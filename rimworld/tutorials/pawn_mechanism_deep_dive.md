# Pawn 機制深度解析：微型模擬系統 (Pawn Deep Dive)

Pawn 是 RimWorld 中最複雜的物件，由數十個相互關聯的子系統組成。

## 1. 組合式架構 (Composition)
Pawn 類別本身不處理邏輯，它是一個「容器」，包含以下核心組件：
*   **`story`**: 身份標識。定義了小人「是誰」（特性、背景、外觀）。
*   **`health`**: 物理狀態。定義了小人「活得怎樣」（傷口、疾病、痛苦）。
*   **`needs`**: 內在驅動。定義了小人「想要什麼」（飢餓、休息、情緒）。
*   **`skills`**: 外部表現。定義了小人「能做什麼」（技能、工作效率）。
*   **`jobs`**: 當前行為。定義了小人「正在做什麼」（Job, Toils）。

## 2. 能力系統 (Pawn Capacities)
這是健康與行為之間的橋樑。所有傷口 (Hediff) 最後都會折算成能力係數：
*   **計算邏輯**: `PawnCapacitiesHandler` 會計算如「意識」、「移動」、「操縱」等數值。
*   **連鎖反應**: 比如「感冒」會降低「意識」，「意識」降低會同時影響「移動」與「操縱」。
*   **臨界點**: 當「意識」或「呼吸」降到 0% 時，Pawn 立即死亡。

## 3. 裝備與庫存
*   **`apparel`**: 穿在身上的衣服。提供防禦力與溫度保護。
*   **`equipment`**: 手拿的武器。提供攻擊用的 `Verb` (動作)。
*   **`inventory`**: 背包。存儲非實體化的 `Thing`。

## 4. 效能優化與 Tick 分級
*   **`Tick()`**: 處理視覺旋轉、移動位移、戰鬥攻擊。
*   **`TickRare()` (250 Ticks)**: 處理傷口癒合、免疫力增長、毒品代謝、心情計算。
*   **`TickLong()` (2000 Ticks)**: 處理長期健康變化、背景社交。

## 5. Mod 開發建議
*   **修改屬性**: 永遠不要直接修改 `Pawn` 類別，應優先修改 `Hediff` (加個狀態) 或 `Stat` (屬性偏移)。
*   **讀取數據**: 使用 `pawn.health.hediffSet.HasHediff(def)` 或 `pawn.GetStatValue(statDef)`。
*   **控制行為**: 透過 `pawn.jobs.StartJob()` 強制小人執行動作。
