# RimWorld 尋路系統：A* 與成本網格 (Pathfinding)

小人如何在大地圖上聰明地繞過障礙物？

## 1. 移動成本 (Path Cost)
尋路的核心公式是：`總成本 = 距離成本 + 地形成本 + 障礙物成本`。
*   **不可通行**: 牆壁、岩石。
*   **可通行但有成本**: 
    *   **地形**: 沙地、雪地會增加移動 Tick。
    *   **物體**: 翻越沙包或桌子會大幅減慢速度。
    *   **動態危險**: 小人會自動避開「火源」所在的格子，除非路徑被強制鎖定。

## 2. 區域系統 (Regions) - 效能的關鍵
*   **Region**: 地圖被劃分為多個 12x12 的區塊。
*   **連通性檢查**: 當你封死一堵牆時，系統會重新計算 Region 的連通性。如果目標區域無法抵達，`PathFinder` 會立即返回失敗，而不會浪費 CPU 去掃描格子。

## 3. 關鍵類別
*   **`Verse.PathFinder`**: 執行 A* 搜尋的核心。
*   **`Verse.Pawn_PathFollower`**: 負責「沿著路徑走」的組件，處理每一步的旋轉與位移。
*   **`Verse.Reachability`**: 一個快速檢查工具，用來判斷「小人能否到達某點」。

## 4. Mod 開發建議
*   **修改移動成本**: 透過 Harmony Patch 攔截 `PathGrid.CalculatedCostAt`。例如：讓你的 Mod 角色在水中走得更快。
*   **強制路徑**: 如果你想讓小人走特定的巡邏路線，可以撰寫自定義的 `JobDriver` 並使用 `Toils_Goto`。
