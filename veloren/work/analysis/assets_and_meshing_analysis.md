# 資產系統與網格優化深度分析 (Analysis)

## 1. 資產加載架構 (`common/assets/src/lib.rs`)
Veloren 使用自定義的加載器來處理多種格式，並支持強大的熱重載功能。

### 核心資產類型：
- **影像 (Image)**：基於 `png/jpg`，轉換為 GPU 紋理。
- **體素 (Voxel)**：加載 `.vox` 檔案，解析出體素網格數據。
- **數據 (RON)**：最核心的格式，用於定義所有遊戲邏輯數值。支持 **MultiRon** 模式，允許外掛 (Plugins) 覆蓋或擴展原始數據。

### 關鍵功能：
- **Asset Tweak**：允許開發者在 `assets/tweak/` 中放置 RON 檔案，實時修改遊戲參數而無需重新編譯。
- **路徑自動尋找**：透過 `ASSETS_PATH` 邏輯，遊戲能自動適應從開發環境、二進位包或系統安裝路徑加載資源。

## 2. 地形與角色網格化 (`voxygen/src/mesh/segment.rs`)
### Greedy Meshing 算法：
- **邏輯**：掃描相鄰體素，若顏色與屬性一致，則將其合併為單個大的 Quad。
- **效能提升**：減少了約 70%-90% 的頂點數量，是 Veloren 能在一般顯示卡上流暢運行的關鍵技術。
- **邊界限制**：單個處理單元限制在 512 立方體素內，以平衡單次生成時間。
