# 尋路精確度、音頻頻道與區域判定深度分析 (Analysis)

## 1. 尋路系統的環境感知精確度 (`common/src/path.rs`)
Veloren 的 NPC 移動並非死板的路徑跟隨，而是具備動態容差。

### 核心技術：
- **動態容差縮放**：[Line 158] 系統會根據環境危險程度縮放 `node_tolerance`。在牆壁或懸崖附近，精確度提升 2 倍以保證安全；在水中，容差放寬 2.5 倍以模擬流體阻力。
- **3D 空間到達判定**：[Line 165] 實作了 XY 平面與 Z 軸的雙重校驗。只有當實體在垂直方向也接近目標點（< 1.0m）時，才會判定為到達路點。
- **障礙物預判**：[Line 140] 透過掃描周圍 8 個方向 (`DIAGONALS`) 的體素屬性，AI 能預判前方的空間開放度，從而決定是否需要切換為「精確導航模式」。

## 2. 沉浸式音訊架構 (`voxygen/src/audio/channel.rs`)
Voxygen 的音訊前端負責處理複雜的背景音樂流轉。

### 關鍵設計：
- **音樂頻道標籤 (`MusicChannelTag`)**：[Line 43] 透過標籤化管理背景音樂。系統支援從「探索」到「戰鬥」的無縫跨步過渡 (Crossfading)。
- **性能化 SFX 管理**：[Line 34] 定義了 `SFX_DIST_LIMIT = 200.0`。這是一個性能邊界，超出 200 米的音效將不再發送給客戶端或由客戶端播放，極大節省了混音資源。
- **循環點定義 (`LoopPoint`)**：[Line 60] 支持自定義音樂循環起始點，這為長達數分鐘的地城 BGM 提供了流暢的重複播放基礎。
