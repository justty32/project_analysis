# Veloren 全方位系統深度拆解：從 RPG 邏輯到渲染引擎

本報告詳盡分析 Veloren 的戰鬥、RPG、伺服器通訊以及視覺技術架構。

## 1. 戰鬥、技能與 RPG 系統

Veloren 的戰鬥不僅是數值計算，而是結合了物理碰撞與動作幀判定的系統。

### 1.1 戰鬥核心機制
- **Poise (失衡值)**：除了生命值 (Health)，戰鬥中還包含霸體判定。遭受重擊會減少 Poise，歸零後會觸發僵直。
- **Combo (連擊)**：記錄成功的連續攻擊次數，用於觸發特定技能效果。
- **能量系統 (Energy)**：施放技能需消耗能量，能量隨時間或成功攻擊而回覆。

### 1.2 技能樹 (`common/src/comp/skillset/`)
- **動態構建**：玩家透過經驗值解鎖技能，系統透過 `skillset_builder.rs` 計算出當前玩家可用的 `Ability` 列表。
- **主動技能 (`ability.rs`)**：定義了技能的「施法前搖、持續時間、後搖、AOE 範圍、傷害係數」。

---

## 2. 裝備與換裝系統

### 2.1 換裝邏輯
- **Voxel 組合**：Veloren 的角色模型是由多個 Voxel Part 組成的。當玩家更換裝備（如頭盔、甲冑）時，系統會動態替換對應部位的 Mesh 數據。
- **染色系統**：支援對特定裝備進行配色，這反映在 Voxel 的頂點顏色數據中。

---

## 3. 內容系統：對話、任務與怪物

### 3.1 對話與任務
- **資源驅動**：對話邏輯主要存放於 `assets/` 中的 RON 檔案。
- **任務系統**：透過 `common/src/event.rs` 捕捉遊戲內的特定行為（如殺死怪物、進入區域），並與玩家狀態連結。

### 3.2 怪物 (Monsters)
- **多樣性**：透過 `common/src/comp/body.rs` 定義基礎形態（鳥、獸、人），再賦予不同的 AI 節點。

---

## 4. 伺服器通訊與資料存檔

### 4.1 玩家資料交換
- **全同步 (Full Sync)**：當玩家加入時，伺服器會將周圍實體（Entity）的快照發送給客戶端。
- **差異同步 (Delta Sync)**：Tick 循環中，伺服器僅廣播發生變動的組件（如：Pos, Vel, Health）。
- **流程**：`client.rs` 封裝協議 -> `network` 傳輸 -> 客戶端 `GlobalState` 更新。

### 4.2 存檔格式 (Save Format)
- **資料庫**：SQLite。
- **角色數據**：存儲在 `characters` 表中，包含等級、技能點、背包（二進位 blob 或 JSON 格式）。
- **轉換層**：使用 `character_loader.rs` 將 SQL 數據轉化為內存中的 `Entity` 組件。

---

## 5. 資源格式與資產管理

- **RON (Rusty Object Notation)**：所有的靜態數據（道具屬性、怪物列表、掉落表、配方）都使用此格式，易於閱讀且與 Rust 類型系統完美對齊。
- **Voxel 格式**：主要是自定義的格式或 `.vox`，用於存儲體素數據。

---

## 6. 視覺與渲染技術

### 6.1 渲染引擎 (`wgpu`)
- **著色器 (Shaders)**：位於 `assets/voxygen/shaders/`。包含地形渲染、水體、光照、後處理效果。
- **延遲渲染管線**：可能採用分層渲染，優先繪製 Opaque (不透明) 物體，再處理透明與光影。

### 6.2 動畫與模型系統
- **Mesh (網格)**：Voxel 會在加載時被「貪婪網格化」(Greedy Meshing)，將相鄰的相同體素合併為一個大的三角形，以減少 GPU 負擔。
- **骨架 (Rigging)**：並非傳統三角形蒙皮，而是「部件層級」的父子節點旋轉/位移。
- **動畫系統 (`voxygen/anim`)**：使用程序化動畫與關鍵幀結合。例如：走路動畫會根據移動速度動態調整步幅。
